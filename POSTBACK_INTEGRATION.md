# Integração de Postback Monetag - Guia de Uso\n\n## 📋 Visão Geral\n\nEsta integração implementa a lógica completa de postback do repositório \"script\" no VoaPix Bot, incluindo:\n\n- ✅ Geração de YMID único por usuário\n- ✅ Verificação de tarefas completas no Young Money\n- ✅ Detecção automática de reset de stats\n- ✅ Interface visual profissional\n- ✅ Polling automático com intervalos configuráveis\n\n---\n\n## 🏗️ Arquitetura\n\n### Backend (`/server`)\n\n#### Arquivos Criados:\n\n1. **`postback.constants.ts`** - Constantes e tipos TypeScript\n   - `POSTBACK_CONFIG` - Configurações (URLs, limites, intervalos)\n   - Interfaces de resposta (MonetagStatsResponse, CheckStatsResponse, etc)\n\n2. **`postback.ts`** - Lógica de negócio\n   - `generateYmid()` - Gera ID único\n   - `fetchMonetagStats(ymid)` - Busca stats da API Monetag\n   - `checkStats(ymid)` - Verifica se atingiu requisitos\n   - `checkReset(ymid)` - Detecta se foi resetado\n   - `isValidYmid(ymid)` - Valida formato do YMID\n\n3. **`routers.ts`** - Rotas tRPC (modificado)\n   - `postback.generateYmid` - Mutation para gerar YMID\n   - `postback.checkStats` - Query para verificar stats\n   - `postback.checkReset` - Query para detectar reset\n\n### Frontend (`/client/src`)\n\n#### Arquivos Criados:\n\n1. **`constants/postback.ts`** - Constantes do cliente\n   - Espelha as constantes do servidor\n   - `CHECK_INTERVAL` = 5000ms (verificação inicial)\n   - `BOT_CHECK_INTERVAL` = 30000ms (verificação de reset)\n\n2. **`hooks/usePostback.ts`** - Hook React para gerenciar postback\n   - Geração de YMID\n   - Polling automático de stats\n   - Detecção de reset\n   - Gerenciamento de timers\n\n3. **`components/PostbackYmidScreen.tsx`** - Componente visual\n   - Tela de verificação YMID\n   - Exibição de stats em tempo real\n   - Botão para acessar Young Money\n   - Indicador de progresso\n\n---\n\n## 🚀 Como Usar\n\n### 1. No Backend\n\n#### Importar as funções de postback:\n\n```typescript\nimport { generateYmid, checkStats, checkReset } from \"./postback\";\n```\n\n#### Usar as rotas tRPC:\n\n```typescript\n// Cliente TypeScript\nconst ymidResult = await api.postback.generateYmid.mutate();\nconst statsResult = await api.postback.checkStats.query({ ymid });\nconst resetResult = await api.postback.checkReset.query({ ymid });\n```\n\n### 2. No Frontend\n\n#### Usar o hook `usePostback`:\n\n```typescript\nimport { usePostback } from \"@/hooks/usePostback\";\n\nfunction MyComponent() {\n  const postback = usePostback();\n\n  // Gerar YMID\n  const handleGenerateYmid = async () => {\n    await postback.generateYmid();\n  };\n\n  return (\n    <div>\n      <p>YMID: {postback.ymid}</p>\n      <p>Impressões: {postback.stats.impressions}</p>\n      <p>Cliques: {postback.stats.clicks}</p>\n      <p>Bot Desbloqueado: {postback.botUnlocked ? \"Sim\" : \"Não\"}</p>\n      <button onClick={handleGenerateYmid}>Gerar YMID</button>\n    </div>\n  );\n}\n```\n\n#### Usar o componente `PostbackYmidScreen`:\n\n```typescript\nimport { PostbackYmidScreen } from \"@/components/PostbackYmidScreen\";\n\nfunction App() {\n  const [botUnlocked, setBotUnlocked] = useState(false);\n\n  return (\n    <>\n      {!botUnlocked ? (\n        <PostbackYmidScreen\n          onBotUnlocked={() => setBotUnlocked(true)}\n          onError={(error) => console.error(error)}\n        />\n      ) : (\n        <div>Bot Screen</div>\n      )}\n    </>\n  );\n}\n```\n\n---\n\n## 🔄 Fluxo de Funcionamento\n\n```\n┌─────────────────────────────────────────────────────────┐\n│ 1. Usuário acessa aplicação                             │\n│    → PostbackYmidScreen é renderizado                   │\n└─────────────────────────────────────────────────────────┘\n                          ↓\n┌─────────────────────────────────────────────────────────┐\n│ 2. usePostback.generateYmid() é chamado                 │\n│    → Backend gera YMID único                            │\n│    → YMID é exibido na tela                             │\n└─────────────────────────────────────────────────────────┘\n                          ↓\n┌─────────────────────────────────────────────────────────┐\n│ 3. Usuário clica \"Acessar Young Money\"                  │\n│    → Abre https://youngmoney.com.br/ em nova aba        │\n│    → Usuário completa tarefas (2 cliques + 20 views)    │\n└─────────────────────────────────────────────────────────┘\n                          ↓\n┌─────────────────────────────────────────────────────────┐\n│ 4. usePostback inicia polling (a cada 5 segundos)       │\n│    → Chama api.postback.checkStats.query({ ymid })      │\n│    → Atualiza stats na tela                             │\n└─────────────────────────────────────────────────────────┘\n                          ↓\n┌─────────────────────────────────────────────────────────┐\n│ 5. Quando stats atingem mínimo (20 imp + 2 cliques)     │\n│    → botUnlocked = true                                 │\n│    → PostbackYmidScreen é escondido                     │\n│    → Bot Screen é exibido                               │\n│    → Polling de reset começa (a cada 30 segundos)       │\n└─────────────────────────────────────────────────────────┘\n                          ↓\n┌─────────────────────────────────────────────────────────┐\n│ 6. Verificação de reset periódica                       │\n│    → Se stats caem abaixo do mínimo                      │\n│    → Se API retorna erro                                │\n│    → Bot é bloqueado novamente                          │\n│    → Volta para tela YMID                               │\n└─────────────────────────────────────────────────────────┘\n```\n\n---\n\n## ⚙️ Configurações\n\n### Constantes de Postback\n\n```typescript\nPOSTBACK_CONFIG = {\n  MONETAG_API: \"https://api.monetag.com/v1/stats\",\n  YOUNG_MONEY_URL: \"https://youngmoney.com.br/\",\n  MIN_IMPRESSIONS: 20,      // Mínimo de impressões\n  MIN_CLICKS: 2,            // Mínimo de cliques\n  CHECK_INTERVAL: 5000,     // 5 segundos - verificação inicial\n  BOT_CHECK_INTERVAL: 30000, // 30 segundos - verificação de reset\n}\n```\n\n### Modificar Configurações\n\nPara alterar os limites ou intervalos:\n\n1. **Backend**: Edite `/server/postback.constants.ts`\n2. **Frontend**: Edite `/client/src/constants/postback.ts`\n\n**Importante**: Manter os valores sincronizados entre backend e frontend!\n\n---\n\n## 📊 Resposta da API Monetag\n\n### Formato Esperado (Sucesso)\n\n```json\n{\n  \"success\": true,\n  \"total_impressions\": 25,\n  \"total_clicks\": 3,\n  \"user_id\": \"ymid_xxxxx\"\n}\n```\n\n### Formato Esperado (Erro/Resetado)\n\n```json\n{\n  \"success\": false,\n  \"message\": \"User not found\"\n}\n```\n\n---\n\n## 🧪 Testando Localmente\n\n### 1. Testar geração de YMID\n\n```bash\ncurl -X POST http://localhost:3000/api/trpc/postback.generateYmid\n```\n\n### 2. Testar verificação de stats\n\n```bash\ncurl -X GET \"http://localhost:3000/api/trpc/postback.checkStats?input=%7B%22ymid%22:%22YMID_1234567890_abc123%22%7D\"\n```\n\n### 3. Testar detecção de reset\n\n```bash\ncurl -X GET \"http://localhost:3000/api/trpc/postback.checkReset?input=%7B%22ymid%22:%22YMID_1234567890_abc123%22%7D\"\n```\n\n---\n\n## 🔍 Monitoramento e Logs\n\n### Logs do Backend\n\nOs erros de conexão com API Monetag são logados em:\n\n```typescript\nconsole.error(\"[Postback] Erro ao buscar stats Monetag:\", error);\n```\n\n### Logs do Frontend\n\nOs erros do hook são logados em:\n\n```typescript\nconsole.error(\"[usePostback] Erro ao verificar stats:\", error);\n```\n\n---\n\n## 🐛 Troubleshooting\n\n### YMID não é gerado\n\n- Verificar se o servidor está rodando\n- Verificar se a rota tRPC está registrada\n- Verificar logs do console\n\n### Stats não atualizam\n\n- Verificar se a API Monetag está acessível\n- Verificar se o YMID é válido\n- Verificar intervalo de polling (CHECK_INTERVAL)\n\n### Bot não desbloqueia\n\n- Verificar se stats atingem o mínimo (20 impressões + 2 cliques)\n- Verificar resposta da API Monetag\n- Verificar console para erros\n\n### Bot bloqueia inesperadamente\n\n- Verificar se stats caíram abaixo do mínimo\n- Verificar se API Monetag retornou erro\n- Verificar intervalo de verificação de reset (BOT_CHECK_INTERVAL)\n\n---\n\n## 📝 Notas Importantes\n\n1. **YMID é único por sessão** - Cada vez que o usuário gera um novo YMID, é um novo rastreamento\n\n2. **Polling é automático** - O hook gerencia todos os timers automaticamente\n\n3. **Cleanup de timers** - Os timers são limpos ao desmontar o componente\n\n4. **Validação de YMID** - O formato é validado com regex: `^YMID_\\d+_[a-z0-9]{6}$`\n\n5. **Tratamento de erros** - Todos os erros são capturados e retornam valores padrão\n\n---\n\n## 🔗 Referências\n\n- **Repositório Original**: https://github.com/Carlos20473736/script\n- **API Monetag**: https://api.monetag.com/v1/stats\n- **Young Money**: https://youngmoney.com.br/\n\n
